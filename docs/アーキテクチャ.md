## 1. 全体構成図

```
┌──────────────────────────────────────────────────────────────────┐
│                     Browser（SPA・React Router 7）                  │
│  App.tsx → AudioProvider → Routes                                │
│    └ Layout（条件付き Navigation / Outlet / GlobalPlayer）          │
│         ├ /              → Gallery                               │
│         ├ /track/:id     → TrackPage（このパスでは Navigation 非表示）│
│         └ /admin         → Admin                                  │
└────────────────────────────┬─────────────────────────────────────┘
                             │ fetch /api/*（開発: Vite プロキシ → :8787）
                             ▼
┌──────────────────────────────────────────────────────────────────┐
│  Express（server/app.ts）                                          │
│  createApp() … ローカル / 単一プロセス本番: API + dist 静的 + SPA   │
│  createApiApp() … Vercel Serverless（静的は配信しない）             │
│  /api/tracks              … Drive カタログ読み取り → JSON           │
│  /api/auth/*              … Google OAuth（Drive 用リフレッシュトークン）│
│  /api/admin/*             … 連携状態・GIS 用設定・アップロード完了   │
│  /api/media/audio|image/:fileId … Drive ストリーム中継（Range 対応）│
│  /api/health              … ヘルスチェック                          │
└────────────────────────────┬─────────────────────────────────────┘
                             │ googleapis（OAuth2 + Drive v3）
                             ▼
                    ┌─────────────────┐
                    │  Google Drive    │
                    │  boss-music-catalog.json │
                    │  音声・画像ファイル      │
                    └─────────────────┘
                             ▲
                    永続化（本番必須: Upstash Redis）
                    リフレッシュトークン / OAuth state / カタログ fileId
```

## 2. デプロイ形態

| 形態 | エントリ | 静的 | API |
|------|-----------|------|-----|
| **ローカル** | `server/index.ts` → `createApp()` | Vite `:3000` | Express `:8787`（`API_PORT` で変更可） |
| **単一プロセス本番** | `NODE_ENV=production` + `tsx server/index.ts` | `dist/` を Express が配信 | 同一プロセス |
| **Vercel** | `api/_handler.ts` を esbuild で `api/index.mjs` にバンドルし、Express `createApiApp()` を default export | `dist/` を CDN／`vercel.json` の rewrite で `index.html` | `/api/*` → Serverless（`maxDuration` 60s） |

`vercel.json` では `/api/(.*)` を `/api` に寄せ、`api/index.mjs` が受ける。Vercel 上ではリクエストパスに `/api` が付かない場合があるため、`createApiApp()` 内で `/api` プレフィックスを補うミドルウェアがある。

## 4. API エンドポイント（概要）

| メソッド | パス | 役割 |
|----------|------|------|
| GET | `/api/tracks` | カタログを読み、公開用 `tracks` を返す（Drive 未設定時は 503 で `tracks: []` 等） |
| GET | `/api/health` | `{ ok: true }` |
| GET | `/api/auth/google` | OAuth 開始（state を Redis 等に保存） |
| GET | `/api/auth/google/callback` | トークン交換・リフレッシュトークン保存・管理者 Cookie |
| GET | `/api/admin/drive-status` | 永続化設定・連携有無の確認 |
| POST | `/api/admin/google-upload-config` | GIS 用 `clientId`・`folderId`・連携ユーザー（要管理者認証） |
| POST | `/api/admin/upload/complete` | Drive 直アップロード後の検証とカタログ追記（要管理者認証） |
| POST | `/api/admin/upload` | **410 Gone**（旧 multipart 経路は廃止） |
| GET | `/api/media/audio/:fileId` | 音声ストリーム |
| GET | `/api/media/image/:fileId` | 画像ストリーム |

## 5. AudioContext 設計（音声状態管理）

### 保持する状態（概念）

- `tracks` … `GET /api/tracks` の結果
- `currentTrackIndex` / `currentTrack`
- `isPlaying` / `currentTime` / `duration` / `volume`
- `isLoading` … 一覧取得中など

### 公開操作

- `play` / `pause` / `resume` / `next` / `prev` / `seek` / `setVolume`

### メディア URL

- `resolveAudioPlaybackUrl`（`mediaUrls.ts`）で、外部 URL・相対パス・`driveAudioFileId` から再生 URL を決定
- `audioCache.ts` で Blob URL 等を扱う

### `<audio>` 要素

- `useRef<HTMLAudioElement>` で 1 要素を保持

| URL | コンポーネント | 説明 |
|-----|----------------|------|

## 8. データ設計（Drive カタログ）

サーバー側の型は `server/services/catalog.ts` の `CatalogRoot` / `TrackRow` が正。Drive 上の JSON は概ね次の形（ファイル名は `CATALOG_FILENAME` = `boss-music-catalog.json`）。

```json
{
  "version": 1,
  "updatedAt": "2026-03-22T12:00:00.000Z",
  "tracks": [
    {
      "id": "track-001",
      "title": "Example",
      "artist": "BOSS × Suno AI",
      "description": "…",
      "createdAt": "2025-03-15",
      "tags": [],
      "playable": true,
      "order": 0,
      "driveAudioFileId": "…",
      "driveCoverFileId": "…"
    }
  ]
}
```

公開応答では `toPublicTrack`（`server/utils/trackPublic.ts`）により `audioUrl` / `coverImage` 等に変換される。

カタログファイル ID は `GOOGLE_DRIVE_CATALOG_FILE_ID`、Redis（`KV_CATALOG_FILE_ID`）、またはローカル `data/catalog-file-id.txt`（`CATALOG_ID_PATH`）のいずれかで保持。
