## 3. AudioContext 設計（音声状態管理）

### 管理する状態（概念）

`AudioContext` は次を保持する（実装は `src/context/AudioContext.tsx` を参照）。

- `tracks` … `GET /api/tracks` で取得した配列
- `currentTrackIndex` / `currentTrack`
- `isPlaying` / `currentTime` / `duration` / `volume`
- `isLoading` … トラック一覧取得中など

### 公開する操作

- `play` / `pause` / `resume` / `next` / `prev` / `seek` / `setVolume`

### メディア URL

- カタログ由来の `audioUrl` が `http(s)` の場合はそのまま、`driveAudioFileId` と組み合わせて `/api/media/audio/:fileId` に解決する場合がある（`mediaUrls.ts`）。
- キャッシュ・Blob URL は `audioCache.ts` で扱う。

### `<audio>` 要素

- `useRef<HTMLAudioElement>` で 1 要素を管理し、ルート遷移で破棄されないようにする（従来方針と同じ）。

## 4. ルーティング設計

| URL | コンポーネント | 説明 |
|-----|----------------|------|
| `/` | `Gallery`（`Layout` 内） | ジャケット一覧 |
| `/track/:id` | `TrackPage` | 曲詳細・再生 |
| `/admin` | `Admin` | 管理（OAuth / アップロード等、要設定） |

`Layout` はトップ以外でナビを隠すなど、パスに応じた UI 分岐を行う（`Layout.tsx`）。

## 5. コンポーネント責務（概要）

| コンポーネント | 責務 |
|----------------|------|
| `App.tsx` | `BrowserRouter` + `AudioProvider` + `Routes` |
| `Layout.tsx` | `Outlet`、ナビの出し分け、`GlobalPlayer` 常時 |

## 6. データ設計（Drive カタログ）

サーバー側の型は `server/services/catalog.ts` の `CatalogRoot` / `TrackRow` が正。概念としては次のような JSON が Drive 上に置かれる。

```json
{
  "version": 1,
  "updatedAt": "2026-03-22T12:00:00.000Z",
  "tracks": [
    {
      "id": "track-001",
      "title": "Example",
      "artist": "BOSS × Suno AI",
      "description": "…",
      "createdAt": "2025-03-15",
      "tags": [],
      "playable": true,
      "order": 0,
      "driveAudioFileId": "…",
      "driveCoverFileId": "…"
    }
  ]
}
```

公開 API 応答では `toPublicTrack` により、`audioUrl` / `coverImage` 等のクライアント向けフィールドに変換される。

## 7. ビジュアル・スタイリング

- **Tailwind CSS v4**: `src/index.css` の `@theme` で `--color-zen-*` やフォント（`Space Grotesk` 等）を定義。
- ユーティリティクラス（`zen-glass`、`landscape-gradient` 等）を `@layer utilities` に集約。

## 8. OGP / メタ情報

SNS向け OGP の動的注入（`react-helmet-async` 等）は **現行コードベースでは未実装**。導入する場合は SSR またはビルド時メタ生成との整理が必要。

## 9. モバイル対応方針

レスポンシブは Tailwind のブレークポイント（`md:` 等）とコンポーネント内クラスで調整。`GlobalPlayer` は画面下部固定。
