## 3. AudioContext 設計（音声状態管理の核心）

### 管理する状態

```js
// AudioContext が保持するグローバル状態
{
  tracks: [...],           // tracks.json の全曲リスト（プレイリスト順）
  currentTrackIndex: 0,    // 現在再生中の曲のインデックス
  isPlaying: false,        // 再生中かどうか
  currentTime: 0,          // 再生位置（秒）
  duration: 0,             // 曲の総時間（秒）
}
```

### 公開する操作（Context の API）

```js
{
  play(trackIndex),   // 指定した曲を再生開始（インデックス指定）
  pause(),            // 一時停止
  resume(),           // 再開
  next(),             // 次の曲へ（最終曲は先頭にループ）
  prev(),             // 前の曲へ
  seek(seconds),      // シーク（プログレスバー操作用）
}
```

### `<audio>` 要素の管理方針

```jsx
// AudioContext.jsx（概念コード）
const audioRef = useRef(new Audio());  // <audio> を React 外で1つだけ生成

// 曲が変わったとき
useEffect(() => {
  audioRef.current.src = tracks[currentTrackIndex].audioUrl;
  audioRef.current.play();
  // URL を該当曲のページに追従
  navigate(`/track/${tracks[currentTrackIndex].id}`, { replace: true });
}, [currentTrackIndex]);

// 曲が終わったとき → 次の曲へ（ループ）
audioRef.current.onended = () => next();
```

> **ポイント**: `<audio>` は React のコンポーネントツリー外（`useRef`）で管理することで、ページ遷移・コンポーネントの再レンダリングによる破棄を防ぐ。

## 4. ルーティング設計

| URL | コンポーネント | 説明 |
|-----|--------------|------|
| `/` | `Gallery.jsx` | ジャケット一覧（トップページ） |
| `/track/:id` | `TrackPage.jsx` | 曲詳細・再生ページ（`id` は `tracks.json` の `id` フィールド） |

### URL 追従の仕組み

```
ユーザーが曲を再生中に [次へ] を押す
  ↓
AudioContext の next() が呼ばれる
  ↓
currentTrackIndex が更新される
  ↓
useEffect が走り audioRef.current.src を次の曲に変更
  ↓
navigate('/track/track-002', { replace: true }) で URL を更新
  ↓
TrackPage が新しい曲の情報を表示（再マウントではなく再レンダリング）
```

## 5. コンポーネント責務一覧

### `App.jsx`
- `AudioProvider` でアプリ全体をラップ
- `<BrowserRouter>` + `<Routes>` でルーティング定義
- `Navigation` と `GlobalPlayer` をルーティングの外側に配置（常時表示）

### `AudioContext.jsx`
- `<audio>` 要素の一元管理
- 再生・停止・曲送りのロジックをすべて集約
- 各コンポーネントへ状態と操作を `useContext` 経由で提供

### `Gallery.jsx`
- `tracks.json` を読み込んでグリッド表示
- `TrackCard` を map で並べる
- レスポンシブグリッド（CSS Grid / Flexbox）

### `TrackCard.jsx`
- ジャケット画像・曲名・「再生可能」バッジを表示
- クリックで `/track/:id` に遷移
- ホバーアニメーション（CSS）

### `TrackPage.jsx`
- URL の `:id` から `tracks.json` で該当曲を検索して表示
- `VinylDisc` に `isPlaying` 状態を渡す
- 再生コントロール（再生・一時停止・次へ・前へ）は `AudioContext` の API を呼ぶだけ

### `VinylDisc.jsx`
- `isPlaying` props を受け取る
- CSS `animation-play-state: running / paused` で回転をコントロール
- props だけで動作する純粋コンポーネント

### `GlobalPlayer.jsx`
- 画面下部に `position: fixed` で常駐
- 現在の曲名・ジャケット・再生コントロール・プログレスバーを表示
- `AudioContext` から状態と操作を取得
- モバイルでは高さを抑えたコンパクトレイアウト

### `Navigation.jsx`
- ロゴ（`/` へのリンク）
- 曲ページではパンくず「ホーム > 曲名」を表示

## 8. レコード回転アニメーション設計

```css
/* VinylDisc.module.css */
@keyframes spin {
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
}

.vinyl {
  animation: spin var(--spin-duration) linear infinite;
  animation-play-state: paused;   /* デフォルトは停止 */
}

.vinyl.playing {
  animation-play-state: running;  /* isPlaying=true のとき running に切り替え */
}
```

```jsx
// VinylDisc.jsx
export default function VinylDisc({ isPlaying, coverImage, alt }) {
  return (
    <img
      src={coverImage}
      alt={alt}
      className={`${styles.vinyl} ${isPlaying ? styles.playing : ''}`}
    />
  );
}
```

## 9. OGP / メタ情報設計

各曲ページで動的に OGP タグを設定する。React Router では `react-helmet-async` を使用。

```jsx
// TrackPage.jsx（概念コード）
import { Helmet } from 'react-helmet-async';

<Helmet>
  <title>{track.title} | Suno Music Portfolio</title>
  <meta property="og:title"       content={track.title} />
  <meta property="og:description" content={track.description} />
  <meta property="og:image"       content={track.coverImage} />
  <meta property="og:url"         content={`https://your-domain.com/track/${track.id}`} />
  <meta property="og:type"        content="music.song" />
</Helmet>
```

> `react-helmet-async` を `devDependencies` に追加する。
