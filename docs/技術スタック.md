# 技術スタック定義

プロジェクト: Suno 楽曲ポートフォリオサイト
作成日: 2026-03-21
対応要件定義: v2

---

## 1. 技術選定一覧

### フロントエンド

| 技術 | バージョン目安 | 役割 | 選定理由 |
|------|--------------|------|----------|
| **React** | 18.x | UI フレームワーク | SPA によるページ遷移時の音声維持。Stitch / AI Studio の出力形式と完全一致 |
| **Vite** | 5.x | ビルドツール | 高速な HMR（開発体験）、軽量な本番バンドル、Firebase Hosting との相性が良い |
| **React Router** | v6.x | クライアントサイドルーティング | フルリロードなしでページ遷移 → `<audio>` 要素を破棄しない設計が実現できる |
| **React Context API** | React 18 組み込み | グローバル状態管理 | 再生状態・プレイリスト管理に使用。Redux 等の外部ライブラリ不要 |

### バックエンド・インフラ（Firebase）

| 技術 | 役割 | 備考 |
|------|------|------|
| **Firebase Hosting** | 静的フロントエンドの配信（HTTPS） | Vite ビルド成果物をそのままデプロイ |
| **Firebase Storage** | MP3 / 画像ファイルのホスティング | フロントと同一プロジェクトのため CORS 設定不要 |
| **Firebase CLI** | ビルド・デプロイの自動化 | `firebase deploy` でフロント＋ストレージをまとめてデプロイ |

> **フェーズ1 では使わない Firebase 機能**（フェーズ2 で拡張用）
> - Firestore: いいね・コメント等のデータ永続化
> - Firebase Authentication: 会員機能
> - Firebase App Hosting: SSR 対応が必要になった場合（現時点は不要）

### スタイリング

| 技術 | 採用タイミング | 備考 |
|------|--------------|------|
| **CSS Modules** または **Tailwind CSS** | Stitch エクスポート後に確定 | Stitch の出力形式に合わせて統一。混在させない |
| **CSS @keyframes** | 実装時 | レコード回転アニメーション用。JS ライブラリ不要 |

### 開発ツール

| ツール | 用途 |
|--------|------|
| **Google Stitch** | UI デザイン・React コード生成 |
| **Google AI Studio（Build 機能）** | ロジック実装・Firebase 統合 |
| **ESLint** | コード品質管理（Vite デフォルト設定） |
| **Prettier** | コードフォーマット統一 |
| **Git / GitHub** | バージョン管理 |

## 2. 依存ライブラリ（`package.json` 想定）

```json
{
  "dependencies": {
    "react": "^18.x",
    "react-dom": "^18.x",
    "react-router-dom": "^6.x"
  },
  "devDependencies": {
    "vite": "^5.x",
    "@vitejs/plugin-react": "^4.x",
    "eslint": "^8.x",
    "eslint-plugin-react": "^7.x",
    "prettier": "^3.x",
    "firebase-tools": "^13.x"
  }
}
```

> **外部 UI ライブラリは原則使わない**: Stitch が UI コンポーネントを生成するため、MUI / Chakra 等との競合を避ける。アニメーションも CSS で完結させる。

|------|-----|------|

## 4. Firebase プロジェクト構成

```
Firebase Project: suno-music-portfolio（仮）
  ├── Hosting
  │     └── / → Vite ビルド成果物（dist/）
  └── Storage
        ├── /audio/      ← MP3 ファイル
        └── /images/     ← ジャケット画像（WebP）
```

### Storage の公開ルール（フェーズ1）

```
// 全ファイルを読み取り公開（認証不要）
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /{allPaths=**} {
      allow read: if true;
      allow write: if false;  // アップロードは Firebase Console から手動
    }
  }
}
```

## 5. 開発・デプロイフロー

```
1. Stitch でUIデザイン
      ↓
2. AI Studio に React コードをエクスポート
      ↓
3. AI Studio でロジック（AudioContext / ルーティング等）を実装
      ↓
4. ローカルで動作確認（vite dev）
      ↓
5. MP3・画像を Firebase Storage にアップロード
      ↓
6. tracks.json の audioUrl / coverImage を Storage の URL に更新
      ↓
7. Firebase Hosting にデプロイ（firebase deploy）
      ↓
8. 本番確認
```
