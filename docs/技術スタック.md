# 技術スタック定義

プロジェクト: Suno 楽曲ポートフォリオサイト（boss-music）
最終更新: 2026-04-24
現行構成: Vercel Blob + Neon Postgres

> このファイルは現行コードと `package.json` を基準にした技術スタック定義。Google Drive / Upstash Redis / Firebase は現在の主構成では使わない。

---

## 1. フロントエンド

| 技術 | 役割 | 備考 |
|------|------|------|
| React 19 | UI フレームワーク | `createRoot`、関数コンポーネント |
| TypeScript 5.8 | 型付き開発 | `npm run lint` は `tsc --noEmit` |
| Vite 6 | 開発サーバー / フロントビルド | `/api` を Express へプロキシ |
| React Router 7 | SPA ルーティング | `BrowserRouter` / nested routes |
| Tailwind CSS 4 | スタイリング | `src/index.css` の `@theme` |
| lucide-react | アイコン | UI ボタン・状態表示 |
| motion | アニメーション | 必要箇所で使用 |
| music-metadata | MP3 ID3 読み取り | 管理画面でタイトル・アーティスト・埋め込みカバー抽出 |

---

## 2. バックエンド / データ

| 技術 | 役割 | 備考 |
|------|------|------|
| Node.js + Express | REST API | `server/app.ts` で `createApp()` / `createApiApp()` を提供 |
| Neon Postgres | 曲メタデータ DB | `tracks` テーブル |
| `@neondatabase/serverless` | Neon 接続 | `server/services/tracksDb.ts` |
| Vercel Blob | 音声・画像ファイル保存 | Public Store。ブラウザ直アップロード |
| `@vercel/blob` | Blob 操作 / client upload | `handleUpload`, `list`, `head`, `del`, `upload` |
| cookie-parser | Cookie 解析 | 管理者セッション Cookie |
| dotenv | ローカル環境変数 | `.env` / `.env.local` |

管理者 API は `ADMIN_SECRET` による `X-Admin-Secret`、または `SESSION_SECRET` 署名 Cookie で保護する。

---

## 3. 開発ツール

| ツール | 用途 |
|--------|------|
| concurrently | `dev:all` で API と Vite を同時起動 |
| wait-on | API 起動待ち |
| tsx | TypeScript のサーバー実行 |
| esbuild | Vercel API bundle `api/index.mjs` 生成 |
| Git | バージョン管理 |

ESLint / Prettier は現行 `package.json` には含まれていない。型チェックは `npm run lint` で行う。

## 6. 環境構成

| 環境 | URL / 接続 | 起動 |
|------|------------|------|
| ローカル frontend | `http://localhost:3000` | `npm run dev` |
| ローカル API | `http://localhost:8787` | `npm run server:dev` |
| ローカル一体 | 上記両方 | `npm run dev:all` |
| Vercel 本番 | `https://boss-music.vercel.app/` | Vercel build / rewrites |

開発時、Vite は `vite.config.ts` の `/api` proxy で Express API に転送する。通常、本番では `VITE_API_BASE_URL` は不要で、相対パス `/api` を使う。

---

## 7. 環境変数

`.env.example` を正とする。

必須:

- `DATABASE_URL`
- `BLOB_READ_WRITE_TOKEN`
- `ADMIN_SECRET`

推奨:

- `SESSION_SECRET`

任意:

- `VITE_ADMIN_SECRET`（ローカル開発用。本番では非推奨）
- `VITE_API_BASE_URL`
- `API_PORT`

---

## 8. データ管理

Neon `tracks` テーブルが曲情報の正本。

主な列:

- `id`
- `title`
- `artist`
- `description`
- `created_at`
- `playlist`
- `tags`
- `playable`
- `sort_order`
- `audio_url`
- `audio_path`
- `audio_size`
- `audio_content_type`
- `cover_url`
- `cover_path`
- `cover_size`
- `cover_content_type`

音声・画像の実体は Vercel Blob に保存し、Neon には Blob URL / pathname / size / content type を保持する。

---

## 9. 現在使わない技術

以下は過去の検討・旧実装・調査ログには登場するが、現行主構成では使わない。

- Firebase Hosting / Storage
- Google Drive API
- Google OAuth
- Upstash Redis
- Drive 上の JSON catalog
- `googleapis`
- `@upstash/redis`
