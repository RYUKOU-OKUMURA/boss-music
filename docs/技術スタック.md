### バックエンド・データ

| 技術 | 役割 | 備考 |
|------|------|------|
| **Node.js + Express** | REST API（`/api/*`） | `server/app.ts` でアプリ生成、`server/index.ts` で listen |
| **tsx** | TypeScript 実行・ウォッチ | `server:dev` / `start` |
| **googleapis** | Google Drive API | カタログ JSON の読み書き、音声・画像のストリーム配信、アップロード後の fileId 検証 |
| **@upstash/redis** | Vercel 本番の永続化 | OAuth リフレッシュトークン・OAuth state・catalog fileId を保存 |
| **cookie-parser** | Cookie 解析 | OAuth セッション用 |
| **dotenv** | 環境変数 | `.env` のあと `.env.local` で上書き（サーバー側） |

楽曲メタデータと Drive 上ファイル ID の一元管理は、**Google ドライブ上のカタログ JSON**（既定ファイル名 `boss-music-catalog.json`）で行う。フロントが静的な `tracks.json` だけを読む構成ではない。

## 3. 環境構成

| 環境 | URL / 接続 | 用途 |
|------|------------|------|
| **ローカル開発（フロント）** | `http://localhost:3000` | `npm run dev`（`vite --port=3000`） |
| **ローカル開発（API）** | `http://localhost:8787` | `npm run server:dev`（`API_PORT` で変更可） |
| **ローカル一体起動** | 上記両方 | `npm run dev:all`（API 起動後に Vite） |

開発時、Vite は `vite.config.ts` の **`/api` → `API_PROXY_TARGET` または `http://127.0.0.1:8787`** のプロキシで API と同一オリジンに見せる（Cookie まわりの都合で `cookieDomainRewrite: ''` を設定）。

| 環境 | 備考 |
|------|------|
| **本番（Vercel）** | `dist/` を静的配信、`/api/*` は `api/index.mjs`（Express `createApiApp`）。MP3 / 画像はブラウザから Google Drive へ直接アップロードし、完了時に `/api/admin/upload/complete` でカタログ更新 |

環境変数のテンプレートは **`.env.example`** を参照（Google OAuth、Drive フォルダ ID、セッション秘密鍵、管理者シークレットなど）。

## 4. Google Drive・カタログまわり（概念）

```
Google Drive（指定フォルダ）
  ├── boss-music-catalog.json   ← 曲一覧・fileId 等（API が読み書き）
  ├── （音声ファイル）
  └── （カバー画像）
```

API は `GET /api/tracks` でカタログを読み、公開用フィールドに整形して返す。メディアは `GET /api/media/audio/:fileId` / `GET /api/media/image/:fileId` で Drive からストリーム中継（Range 対応）。

管理アップロードは次の 2 段階:

1. `POST /api/admin/google-upload-config` で Google クライアント設定と連携済みアカウント情報を取得
2. 管理画面が Google Identity Services で Google Drive 用アクセストークンを取得
3. ブラウザが Google Drive に直接アップロードし、最後に `POST /api/admin/upload/complete` で catalog を更新

Vercel 本番では `Upstash Redis` を必須とし、ローカルファイルへの永続化フォールバックは無効。

## 5. 開発・デプロイフロー（現行）

```
1. .env / .env.local を用意（.env.example 参照）
2. Google Cloud で OAuth クライアント・Drive API を有効化し、フォルダ ID を設定
3. npm install
4. npm run dev:all  →  API :8787 + Vite :3000
5. ブラウザで動作確認（管理画面は /admin、必要に応じて OAuth）
6. Vercel 本番は Upstash Redis を接続し、Google Drive OAuth 完了後に管理画面から MP3 / 画像を直接 Drive へ送信（`npm run build` で `dist/` と `api/index.mjs` を生成）
7. 単一プロセス本番は `npm run build` → `NODE_ENV=production` で `tsx server/index.ts`（またはプロセスマネージャ＋リバースプロキシ）
```
