# Suno 楽曲ポートフォリオサイト — 要件定義 v2

作成日: 2025-03-21
最終更新: 2026-03-21
ステータス: レビュー済み（実装準備完了）

---

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| **目的** | Suno AI で制作した楽曲を「いつでも聴ける」形で公開し、作品ポートフォリオとして見せる |
| **コンセプト** | ディスコ／クラブ風の暗めの世界観の中で、ジャケットを軸に楽曲を探索・再生できる Web サイト |
| **音声形式** | MP3（`.mp3`） |

---

## 2. 技術スタック

| 区分 | 採用技術 | 選定理由 |
|------|----------|----------|
| **フレームワーク** | React 18+（Vite） | SPA によるシームレスな音声再生。Stitch / AI Studio との親和性が最も高い |
| **ルーティング** | React Router v6 | ページ遷移時も `<audio>` を破棄しないクライアントサイドルーティング |
| **状態管理** | React Context | グローバルな再生状態（AudioContext）の管理。外部ライブラリ不要の軽量構成 |
| **スタイリング** | Stitch 出力に準拠（CSS Modules or Tailwind CSS） | Stitch からのエクスポート形式に合わせて実装時に確定 |
| **データ形式** | JSON（`tracks.json`） | 曲追加が容易。CMS 不要でシンプル |
| **音声ホスティング** | Firebase Storage（Cloud Storage） | AI Studio との統合でセットアップ容易。CORS 設定も一元管理 |
| **ホスティング** | Firebase Hosting | HTTPS 標準対応。Firebase Storage と同一プロジェクトで完結 |

### 開発ワークフロー

```
Stitch（UIデザイン）
  ↓ React コードをエクスポート
Google AI Studio（ロジック実装・Firebase 統合）
  ↓ ビルド＆デプロイ
Firebase Hosting + Storage（本番環境）
```

## 3. ユーザーストーリー

1. **訪問者**として、トップで作品一覧を**視覚的にざっと把握**したい（レコードジャケットのギャラリー）。
2. **訪問者**として、**どの曲が再生できるか**を迷わず知りたい。
3. **訪問者**として、気になるジャケットをクリックして**その曲専用のページ**に行き、そこで再生したい（**1ページ1曲**）。
4. **訪問者**として、一度再生を始めたら**プレイリストとして連続再生**したい。
5. **訪問者**として、再生ボタンで音楽を聴き、**再生中であることが視覚でも分かる**（レコードが回る）ようにしたい。
6. **訪問者**として、**ページを移動しても再生中の音楽が途切れない**ようにしたい。

## 4. 機能要件

### 4.1 トップページ（ギャラリー）

- **レコードジャケット風のカード／タイル**を多数並べるギャラリー表示（グリッド想定）。
- 各カードに含める情報（最低限）:
  - **カバー画像（アイキャッチ）**
  - **曲名**（短いラベル）
  - **「再生可能」の明示**（バッジ、アイコン、色分け、オーバーレイ文言など、**一目で区別できる UI**）
- **クリック**で「その曲の再生ページ」へ遷移。
- （任意）ホバーで軽い演出（光彩・ズームなど）、クラブ／ディスコの雰囲気に合うマイクロインタラクション。

### 4.2 楽曲詳細（再生）ページ — 1ページ1曲

- **1曲につき 1 URL／1 画面**で、その曲の情報と再生を完結させる。
- **同じジャケット画像**を大きく表示（トップと一貫）。
- **再生コントロール**: 再生／一時停止、音量、次へ／前へ（プレイリスト連携）。
- **再生中のフィードバック**:
  - 再生中は**レコード（またはジャケット円盤）が回転アニメーション**。
  - 一時停止／停止時は回転停止（または減速）で状態が分かるようにする。
- **曲情報**（例）: タイトル、制作日やメモ、短いコメント、クレジット — 運用で詳細化。

### 4.3 プレイリスト（連続再生）— 方式A: グローバルミニプレイヤー

**採用方式**: サイト共通のミニプレイヤー（画面下部固定）で 1 つの `<audio>` を管理する**方式A**を採用。

- **グローバルミニプレイヤー**がアプリ全体（`App.jsx`）に常駐し、ページ遷移で破棄されない。
- ユーザーが**いずれかの曲から再生を開始**したあと、**定義された順序**で MP3 が途切れず次々に再生される。
- **再生順**: `tracks.json` の配列順 = ギャラリーの並び順 = 再生順とする。
- **URL 追従**: 次曲に自動で進んだ際、`history.pushState` によりブラウザの URL を該当曲のページに更新する（共有・ブックマーク対応）。
- **最終曲のあと**: **先頭にループ**して連続再生を継続する（クラブ的な世界観に合致）。
- ミニプレイヤーには最低限: 曲名、再生／一時停止、次へ／前へ、プログレスバー。

### 4.4 ナビゲーション・全体

- トップ ↔ 各曲ページで**戻る／パンくず／ロゴでホーム**のいずれかで迷子にならないこと。
- **モバイル**でもギャラリーと再生 UI が実用的であること（レスポンシブ）。
- モバイルではミニプレイヤーがコンテンツと干渉しないよう、下部に十分な余白を確保する。

## 5. 非機能要件

| 区分 | 内容 |
|------|------|
| **ビジュアル** | クラブ／ディスコ寄り、**暗めの背景**、ネオン・メタリック・グラデーションなど「かっこいい」トーン。読みやすさとのコントラスト両立。 |
| **パフォーマンス** | 初回表示が極端に重くならない（画像最適化、遅延読み込みの検討）。MP3 はストリーミング再生（全体ダウンロード待ちにしない）。 |
| **アクセシビリティ** | キーボード操作、代替テキスト、コントラストの最低限の配慮。 |
| **ブラウザ** | Chrome / Safari / Firefox / Edge の最新版で MP3 再生が可能であること。 |
| **配信** | Firebase Hosting による **HTTPS** 配信。音声は Firebase Storage から配信（同一プロジェクトのため CORS 設定不要）。 |
| **OGP / メタ情報** | 各曲ページに `og:title`、`og:image`（ジャケット画像）、`og:url` を設定し、SNS シェア時にカード表示されるようにする。 |
| **自動再生ポリシー** | ブラウザの自動再生制限により、**初回再生はユーザー操作（クリック）を必須**とする。連続再生（2曲目以降）はユーザー操作後のセッション内であれば自動で再生可能。 |

## 6. コンテンツ・データ

### 6.1 楽曲データ構造（`tracks.json`）

```json
{
  "tracks": [
    {
      "id": "track-001",
      "title": "曲名",
      "artist": "BOSS × Suno AI",
      "description": "曲の短い説明やメモ",
      "createdAt": "2025-03-15",
      "audioUrl": "/audio/track-001.mp3",
      "coverImage": "/images/track-001.webp",
      "playable": true
    }
  ]
}
```

### 6.2 画像仕様

| 項目 | 仕様 |
|------|------|
| **アスペクト比** | 1:1（正方形、レコードジャケットの慣例） |
| **推奨サイズ** | 800×800px（トップ用サムネイルはCSS縮小） |
| **フォーマット** | WebP 推奨（フォールバック用に JPEG も可） |
| **OGP 用** | 1200×630px の別画像、または正方形画像をそのまま使用 |

### 6.3 クレジット表記

- サイトフッターに「Music created with Suno AI」の共通クレジットを配置。
- 各曲ページの曲情報エリアにも個別クレジット欄を設ける（任意入力）。

## 7. アプリケーション構成

```
src/
  components/
    GlobalPlayer.jsx      ← 画面下部固定のミニプレイヤー
    Gallery.jsx            ← トップのジャケット一覧グリッド
    TrackCard.jsx          ← ギャラリー内の個別カード
    TrackPage.jsx          ← 1曲1ページの詳細・再生画面
    VinylDisc.jsx          ← 回転するレコードアニメーション
    Navigation.jsx         ← ヘッダー／パンくず
  context/
    AudioContext.jsx        ← 再生状態・プレイリスト・現在曲の管理
  data/
    tracks.json             ← 楽曲データ
  App.jsx                   ← ルーティング定義 + AudioProvider + GlobalPlayer
  main.jsx                  ← エントリポイント
public/
  audio/                    ← MP3 ファイル（または Firebase Storage URL を使用）
  images/                   ← ジャケット画像
```

## 8. 技術メモ

- **再生エンジン**: HTML5 `<audio>` 要素を `AudioContext`（React Context）で一元管理。コンポーネントのマウント／アンマウントに依存しない設計。
- **アニメーション**: CSS `@keyframes` + `animation-play-state` で「再生中のみ回転」を実現。JS 側から `playing` / `paused` の状態を CSS クラスで切り替え。
- **URL 追従**: React Router の `useNavigate` + `window.history.pushState` で、曲の自動遷移時に URL を同期。
- **画像最適化**: ギャラリーでは `loading="lazy"` による遅延読み込み。初期表示のファーストビューに入る画像のみ即時読み込み。

## 10. 受け入れ条件（完了の定義）

1. トップで**複数曲がジャケットギャラリー**として見え、**再生可能な曲が視覚的に識別できる**。
2. ジャケットをクリックすると**該当曲の専用ページ（1ページ1曲）**に遷移し、そのページで**当該 MP3 を再生**できる。
3. **グローバルミニプレイヤー**が画面下部に常駐し、**ページ遷移しても再生が途切れない**。
4. **プレイリストとして定義された複数 MP3** が**連続再生**でき、最終曲後は先頭にループする。
5. 再生ページで**再生するとレコードが回転**し、停止時に回転が止まって状態が視覚で分かる。
6. 全体のトーンが**暗め・クラブ／ディスコ風**で統一されている。
7. **モバイル端末**でギャラリー閲覧・楽曲再生・ミニプレイヤー操作が実用的に行える。
8. 各曲ページの URL を SNS でシェアした際、**OGP カード（タイトル＋ジャケット画像）が表示**される。
